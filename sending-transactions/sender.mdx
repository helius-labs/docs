## Overview

Helius Sender is a specialized service designed for high-frequency traders and applications requiring ultra-low latency transaction submission. This service provides geographically distributed endpoints with optimized routing to ensure your transactions reach the Solana network as quickly as possible.

Sender optimizes transaction latency by sending to both Solana validators and Jito's infrastructure simultaneously, providing multiple pathways for your transactions to be included in blocks.

**Key Features:**
- **Dual Routing**: Sends to both validators and Jito infrastructure for optimal speed
- **Global Distribution**: Multiple geographic endpoints for optimal routing  
- **No Credit Usage**: Available on all plans without consuming API credits
- **High Throughput**: Default 5 TPS with option to increase via support ticket

## Usage

The Sender endpoint uses the same `sendTransaction` method as standard RPC endpoints but with specific requirements for optimal performance. **All transactions must include a minimum 0.001 SOL tip and skip preflight checks.**

### Basic Request Format

```typescript
{
  "id": "unique-request-id",
  "jsonrpc": "2.0", 
  "method": "sendTransaction",
  "params": [
    "BASE64_ENCODED_TRANSACTION", // Must include 0.001 SOL tip instruction
    {
      "encoding": "base64",
      "skipPreflight": true,       // Required: must be true
      "maxRetries": 0
    }
  ]
}
```

<Warning>
The `BASE64_ENCODED_TRANSACTION` above must contain a SOL transfer instruction with a minimum 0.001 SOL tip to one of the designated tip accounts. Without this tip, your transaction will be rejected.
</Warning>

## Requirements

### 1. Skip Preflight (Mandatory)

The `skipPreflight` parameter **must** be set to `true`. This endpoint is optimized for traders who prioritize speed over transaction validation.

```typescript
{
  "skipPreflight": true  // Required: must be true
}
```

<Warning>
Since preflight checks are skipped, ensure your transactions are properly constructed and funded before submission. Invalid transactions will be rejected by the network after submission.
</Warning>

### 2. Minimum Tip Requirement

All transactions submitted through Sender **must include a SOL transfer with a minimum tip of 0.001 SOL** to one of the designated tip accounts. This requirement exists because Sender routes your transactions through both regular Solana validators and Jito infrastructure simultaneously.

#### Why Tips Are Required

Tips serve multiple purposes in Sender's dual routing system:

1. **Jito Infrastructure Access**: Jito infrastructure uses tips to determine transaction priority in their block building processes  
2. **Network Efficiency**: Tips help reduce spam and ensure only serious transactions use the fast routing service
3. **Validator Incentives**: Tips compensate validators for processing your transaction with higher priority

#### How Tips Work

When you include a tip in your transaction:
- **Jito Infrastructure**: Uses tips in their auction mechanism to determine which transactions get priority placement in blocks
- **Dual Benefit**: Since Sender sends to both systems simultaneously, your tipped transaction has multiple pathways to be included quickly

#### Tip Amount Guidelines

- **Minimum**: 0.001 SOL (required for Sender to accept your transaction)
- **Recommended**: Slightly above minimum (0.0015-0.002 SOL) for better prioritization during network congestion
- **High Priority**: During peak congestion, consider 0.005+ SOL for maximum priority

## Complete Implementation Example

Here's a full example showing how to construct and send a transaction with Sender including proper tipping:

```typescript
import { 
  Connection, 
  Transaction, 
  SystemProgram, 
  PublicKey,
  Keypair,
  LAMPORTS_PER_SOL
} from '@solana/web3.js';

async function sendWithSender() {
  // Your transaction logic here
  const senderKeypair = Keypair.fromSecretKey(/* your secret key */);
  const connection = new Connection('https://mainnet.helius-rpc.com/?api-key=YOUR_API_KEY');
  
  // Get recent blockhash
  const { blockhash } = await connection.getLatestBlockhash();
  
  // Create transaction with your main logic
  const transaction = new Transaction();
  
  // Add your main transaction instructions here
  // transaction.add(yourMainInstruction);
  
  // Add required tip (minimum 0.001 SOL)
  const tipAccount = new PublicKey("4ACfpUFoaSD9bfPdeu6DBt89gB6ENTeHBXCAi87NhDEE");
  const tipAmount = 0.001 * LAMPORTS_PER_SOL; // 0.001 SOL minimum
  
  transaction.add(
    SystemProgram.transfer({
      fromPubkey: senderKeypair.publicKey,
      toPubkey: tipAccount,
      lamports: tipAmount,
    })
  );
  
  // Set recent blockhash and sign
  transaction.recentBlockhash = blockhash;
  transaction.sign(senderKeypair);
  
  // Serialize transaction
  const serializedTx = transaction.serialize().toString('base64');
  
  // Send via Sender endpoint
  const senderRequest = {
    "id": Date.now().toString(),
    "jsonrpc": "2.0",
    "method": "sendTransaction",
    "params": [
      serializedTx,
      {
        "encoding": "base64",
        "skipPreflight": true,
        "maxRetries": 0
      }
    ]
  };
  
  // Choose optimal endpoint based on your location
  const endpoint = 'http://slc-sender.helius-rpc.com/fast';
  
  try {
    const response = await fetch(endpoint, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(senderRequest)
    });
    
    const result = await response.json();
    
    if (result.error) {
      console.error('Transaction failed:', result.error);
      return null;
    }
    
    console.log('Transaction sent successfully:', result.result);
    return result.result; // Transaction signature
    
  } catch (error) {
    console.error('Request failed:', error);
    return null;
  }
}
```

## Best Practices

### Endpoint Selection
Choose the endpoint closest to your infrastructure:
- **US West**: Use `slc-sender` (Salt Lake City)  
- **US East**: Use `ewr-sender` (Newark) or `pitt-sender` (Pittsburgh)
- **Europe**: Use `fra-sender` (Frankfurt) or `ams-sender` (Amsterdam)
- **Asia**: Use `sg-sender` (Singapore) or `tyo-sender` (Tokyo)

### Transaction Construction
1. **Minimize Transaction Size**: Smaller transactions propagate faster
2. **Optimal Tip Amount**: Use slightly above minimum (0.001 SOL) for better prioritization
3. **Recent Blockhash**: Always use the most recent blockhash possible
4. **Proper Signing**: Ensure all required signatures are present

### Error Handling
```typescript
async function handleSenderTransaction(serializedTx: string) {
  const endpoints = [
    'http://slc-sender.helius-rpc.com/fast',
    'http://ewr-sender.helius-rpc.com/fast', // Fallback
  ];
  
  for (const endpoint of endpoints) {
    try {
      const response = await fetch(endpoint, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          id: Date.now().toString(),
          jsonrpc: "2.0",
          method: "sendTransaction",
          params: [serializedTx, { 
            encoding: "base64", 
            skipPreflight: true, 
            maxRetries: 0 
          }]
        })
      });
      
      const result = await response.json();
      
      if (!result.error) {
        return result.result;
      }
      
      console.warn(`Endpoint ${endpoint} failed:`, result.error);
      
    } catch (error) {
      console.warn(`Endpoint ${endpoint} unreachable:`, error);
      continue; // Try next endpoint
    }
  }
  
  throw new Error('All Sender endpoints failed');
}
```

## Monitoring and Confirmation

Since Sender transactions skip preflight checks, implement proper monitoring:

```typescript
async function monitorTransaction(signature: string) {
  const connection = new Connection('https://mainnet.helius-rpc.com/?api-key=YOUR_API_KEY');
  
  // Wait for confirmation
  const confirmation = await connection.confirmTransaction(signature, 'confirmed');
  
  if (confirmation.value.err) {
    console.error('Transaction failed:', confirmation.value.err);
    return false;
  }
  
  console.log('Transaction confirmed:', signature);
  return true;
}

// Usage
const signature = await sendWithSender();
if (signature) {
  await monitorTransaction(signature);
}
```
