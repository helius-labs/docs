## Overview

Helius Sender is a specialized service for ultra-low latency transaction submission. It optimizes transaction latency by sending to both Solana validators and Jito infrastructure simultaneously, providing multiple pathways for your transactions to be included in blocks.

## Requirements

<Warning>
**Mandatory Requirements**: All transactions must include a minimum 0.001 SOL tip and skip preflight checks.
</Warning>

### 1. Skip Preflight (Mandatory)

The `skipPreflight` parameter **must** be set to `true`. Sender is optimized for traders who prioritize speed over transaction validation.

```typescript
{
  "skipPreflight": true  // Required: must be true
}
```

<Warning>
Since preflight checks are skipped, ensure your transactions are properly constructed and funded before submission. Invalid transactions will be rejected by the network after submission.
</Warning>

### 2. Minimum Tip Requirement

All transactions submitted through Sender **must include a SOL transfer with a minimum tip of 0.001 SOL** to one of the designated tip accounts. This requirement exists because Sender routes your transactions through both regular Solana validators and Jito infrastructure simultaneously.

#### Why Tips Are Required

Tips serve multiple purposes in Sender's dual routing system:

1. **Transaction Prioritization**: Tips signal to validators that your transaction should be prioritized over others in the mempool
2. **Jito Infrastructure Access**: Jito infrastructure uses tips to determine transaction priority in their block building processes  
3. **Network Efficiency**: Tips help reduce spam and ensure only serious transactions use the fast routing service
4. **Validator Incentives**: Tips compensate validators for processing your transaction with higher priority

#### How Tips Work

When you include a tip in your transaction:
- **Regular Validators**: See the tip as a priority fee signal and are more likely to include your transaction in their blocks
- **Jito Infrastructure**: Uses tips in their auction mechanism to determine which transactions get priority placement in blocks
- **Dual Benefit**: Since Sender sends to both systems simultaneously, your tipped transaction has multiple pathways to be included quickly

#### Tip Amount Guidelines

- **Minimum**: 0.001 SOL (required for Sender to accept your transaction)
- **Recommended**: Slightly above minimum (0.0015-0.002 SOL) for better prioritization during network congestion
- **High Priority**: During peak congestion, consider 0.005+ SOL for maximum priority

## Usage

The Sender endpoint uses the same `sendTransaction` method as standard RPC endpoints but with specific requirements for optimal performance. **All transactions must include a minimum 0.001 SOL tip and skip preflight checks.**

### Basic Request Format

```typescript
{
  "id": "unique-request-id",
  "jsonrpc": "2.0", 
  "method": "sendTransaction",
  "params": [
    "BASE64_ENCODED_TRANSACTION", // Must include 0.001 SOL tip instruction
    {
      "encoding": "base64",
      "skipPreflight": true,       // Required: must be true
      "maxRetries": 0
    }
  ]
}
```

<Warning>
The `BASE64_ENCODED_TRANSACTION` above must contain a SOL transfer instruction with a minimum 0.001 SOL tip to one of the designated tip accounts. Without this tip, your transaction will be rejected.
</Warning>

### Complete Implementation Example

```typescript
import { 
  Connection, 
  Transaction, 
  SystemProgram, 
  PublicKey,
  Keypair,
  LAMPORTS_PER_SOL,
  ComputeBudgetProgram
} from '@solana/web3.js';

async function sendWithSender() {
  const senderKeypair = Keypair.fromSecretKey(/* your secret key */);
  const connection = new Connection('https://mainnet.helius-rpc.com/?api-key=YOUR_API_KEY');
  
  // 1. Get recent blockhash with proper commitment
  const { blockhash, lastValidBlockHeight } = await connection.getLatestBlockhash('confirmed');
  
  // 2. Create transaction with your main instructions
  const transaction = new Transaction();
  
  // Add your main transaction instructions here
  // transaction.add(yourMainInstruction);
  
  // 3. Add required tip (minimum 0.001 SOL)
  const tipAccount = new PublicKey("4ACfpUFoaSD9bfPdeu6DBt89gB6ENTeHBXCAi87NhDEE");
  const tipAmount = 0.001 * LAMPORTS_PER_SOL;
  
  transaction.add(
    SystemProgram.transfer({
      fromPubkey: senderKeypair.publicKey,
      toPubkey: tipAccount,
      lamports: tipAmount,
    })
  );
  
  // 4. Optimize compute units (optional but recommended)
  transaction.add(
    ComputeBudgetProgram.setComputeUnitLimit({ units: 300_000 }) // Adjust based on your needs
  );
  
  // 5. Set recent blockhash and sign
  transaction.recentBlockhash = blockhash;
  transaction.sign(senderKeypair);
  
  // 6. Serialize transaction
  const serializedTx = transaction.serialize().toString('base64');
  
  // 7. Send via Sender endpoint
  const senderRequest = {
    "id": Date.now().toString(),
    "jsonrpc": "2.0",
    "method": "sendTransaction",
    "params": [
      serializedTx,
      {
        "encoding": "base64",
        "skipPreflight": true,       // Required
        "maxRetries": 0              // Implement your own retry logic
      }
    ]
  };
  
  // Choose optimal endpoint based on your location
  const endpoint = 'http://slc-sender.helius-rpc.com/fast';
  
  try {
    const response = await fetch(endpoint, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(senderRequest)
    });
    
    const result = await response.json();
    
    if (result.error) {
      console.error('Transaction failed:', result.error);
      return null;
    }
    
    console.log('Transaction sent successfully:', result.result);
    
    // 8. Monitor confirmation
    await monitorTransaction(result.result, connection, lastValidBlockHeight);
    
    return result.result;
    
  } catch (error) {
    console.error('Request failed:', error);
    return null;
  }
}

// Proper confirmation monitoring
async function monitorTransaction(
  signature: string, 
  connection: Connection, 
  lastValidBlockHeight: number
) {
  let blockHeight = await connection.getBlockHeight();
  
  while (blockHeight < lastValidBlockHeight) {
    const confirmation = await connection.confirmTransaction({
      signature,
      blockhash: transaction.recentBlockhash,
      lastValidBlockHeight
    }, 'confirmed');
    
    if (confirmation.value.err) {
      throw new Error(`Transaction failed: ${confirmation.value.err}`);
    }
    
    if (confirmation.value.confirmationStatus === 'confirmed') {
      console.log('Transaction confirmed:', signature);
      return true;
    }
    
    // Wait before checking again
    await new Promise(resolve => setTimeout(resolve, 1000));
    blockHeight = await connection.getBlockHeight();
  }
  
  throw new Error('Transaction expired');
}
```

### Dynamic Tip Amounts

```typescript
// Adjust tip based on network conditions
const getTipAmount = (networkCongestion: 'low' | 'medium' | 'high') => {
  const tipAmounts = {
    'low': 0.001,      // Minimum required
    'medium': 0.002,   // Better priority
    'high': 0.005      // Maximum priority
  };
  return tipAmounts[networkCongestion] * LAMPORTS_PER_SOL;
};

// Usage
const tipAmount = getTipAmount('medium');
```

## Best Practices

### Endpoint Selection
Choose the endpoint closest to your infrastructure:
- **US West**: `slc-sender` (Salt Lake City)  
- **US East**: `ewr-sender` (Newark) or `pitt-sender` (Pittsburgh)
- **Europe**: `fra-sender` (Frankfurt) or `ams-sender` (Amsterdam)
- **Asia**: `sg-sender` (Singapore) or `tyo-sender` (Tokyo)

For optimal latency, consider co-locating with Helius servers in Frankfurt or Pittsburgh.

### Transaction Setup
- Use `skipPreflight: true` and `maxRetries: 0`
- Include minimum 0.001 SOL tip to designated accounts
- Fetch blockhash with `'confirmed'` commitment
- Implement your own retry logic
- Set appropriate compute unit limits

## Rate Limits and Scaling

- **Default Rate Limit**: 5 transactions per second
- **No Credit Usage**: Sender transactions don't consume API credits from your plan
