## Standard Archival Methods

Access historical blockchain data with these standard RPC methods.

### Block Data Methods

Query historical block information and metadata.

#### getBlock

Retrieve complete block data including all transactions and metadata for a specific slot.

**Use cases:**
- Block explorers
- Audit tools
- Historical analysis

<Card title="View getBlock guide" icon="book-open" href="/rpc/guides/getblock" horizontal />

#### getBlocks

Get a list of confirmed blocks between two slots.

**Use cases:**
- Gap detection
- Block range analysis
- Indexing

<Card title="View getBlocks guide" icon="book-open" href="/rpc/guides/getblocks" horizontal />

#### getBlockTime

Get the estimated production time of a block as Unix timestamp.

**Use cases:**
- Time-based analytics
- Event correlation

<Card title="View getBlockTime guide" icon="book-open" href="/rpc/guides/getblocktime" horizontal />

### Transaction History Methods

Access historical transaction data and signatures.

#### getTransaction

Retrieve full transaction details for a specific signature.

**Use cases:**
- Transaction verification
- Detailed analysis
- Debugging

<Card title="View getTransaction guide" icon="book-open" href="/rpc/guides/gettransaction" horizontal />

#### getSignaturesForAddress

Get confirmed signatures for transactions involving an address.

**Use cases:**
- Basic transaction history
- Signature collection

<Note>
For advanced filtering, sorting, and token account history, use [`getTransactionsForAddress`](/rpc/gettransactionsforaddress) instead. Note that `getSignaturesForAddress` does not include transactions involving associated token accounts.
</Note>

<Card title="View getSignaturesForAddress guide" icon="book-open" href="/rpc/guides/getsignaturesforaddress" horizontal />

#### getTransactionsForAddress

Enhanced transaction history with advanced filtering, bidirectional sorting, token account support, and efficient pagination. **Costs 100 credits per request.**

**Use cases:**
- Complete wallet token history (including associated token accounts)
- Time-based transaction analysis
- Status filtering (succeeded/failed)
- Chronological historical replay
- Complete transaction data in one call

<Card title="View getTransactionsForAddress guide" icon="book-open" href="/rpc/gettransactionsforaddress" horizontal />

#### getInflationReward

Get inflation reward for a list of addresses for an epoch.

**Use cases:**
- Validator rewards
- Staking analytics

<Card title="View getInflationReward guide" icon="book-open" href="/rpc/guides/getinflationreward" horizontal />

## What historical data is available?

Helius provides complete historical data from Solana genesis to present, continuously updated in real-time. Our indexed and structured storage enables fast retrieval, allowing you to query years of on-chain activity efficiently.

All historical data includes full support for versioned transactions and all Solana program types. Helius maintains this data on independent archival infrastructure, delivering consistent performance without the bottlenecks or reliability issues common to shared BigTable systems.
