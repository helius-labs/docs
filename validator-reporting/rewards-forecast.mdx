## Endpoint

```
GET http://validator-api.helius-internal.com/validators/{vote_account}/rewards/forecast
```

## Description

Generate rewards forecasts based on historical performance data using statistical models. The API provides three scenario projections (conservative, baseline, optimistic) with confidence intervals and detailed methodology information.

### Forecasting Models

<CardGroup cols={2}>
  <Card title="Rolling Average" icon="chart-simple">
    Uses mean ± standard deviation from historical data.
  </Card>
  <Card title="Linear Regression" icon="chart-line">
    Extrapolates trends from historical data.
  </Card>
</CardGroup>

### APY Calculation Formula

```
APY = ((1 + epoch_return)^num_epochs_per_year - 1) * 100

Where num_epochs_per_year ≈ 182.5
```

## Path Parameters

<ParamField path="vote_account" type="string" required>
  The validator's vote account public key (base58 encoded)

  Must match pattern: `^[1-9A-HJ-NP-Za-km-z]{32,44}$`

  Example: `he1iusunGwqrNtafDtLdhsUQDFvo13z9sUa36PauBtk`
</ParamField>

---

## Query Parameters

<ParamField query="forecast_period" type="string" default="30d">
  Time period to forecast into the future

  Options: `7d`, `30d`, `90d`, `180d`, `1y`
</ParamField>

<ParamField query="lookback_days" type="integer" default="30">
  Number of historical days to analyze for forecasting

  Range: 7-365 days

  Longer lookback periods provide more stable forecasts but may miss recent trends.
</ParamField>

<ParamField query="model" type="string" default="rolling_average">
  Forecasting model to use

  Options:
  - `rolling_average`: Mean ± standard deviation approach
  - `linear_regression`: Trend extrapolation
</ParamField>

<ParamField query="include_breakdown" type="boolean" default="false">
  Include detailed breakdown of rewards by type (inflation, MEV, block rewards)
</ParamField>

```javascript JavaScript
const getRewardsForecast = async (voteAccount, options = {}) => {
  const baseUrl = 'http://validator-api.helius-internal.com';

  const params = new URLSearchParams({
    forecast_period: options.forecastPeriod || '30d',
    lookback_days: options.lookbackDays || 30,
    model: options.model || 'rolling_average',
    include_breakdown: options.includeBreakdown || false
  });

  const url = `${baseUrl}/validators/${voteAccount}/rewards/forecast?${params}`;
  const response = await fetch(url);
  const data = await response.json();

  return data;
};

// Usage - Rolling Average
const forecast = await getRewardsForecast(
  'he1iusunGwqrNtafDtLdhsUQDFvo13z9sUa36PauBtk',
  {
    forecastPeriod: '30d',
    lookbackDays: 30,
    model: 'rolling_average',
    includeBreakdown: true
  }
);

console.log('Baseline APY:', forecast.projections.baseline.apy_pct, '%');
console.log('Baseline SOL:', forecast.projections.baseline.rewards.total_sol);
console.log('Confidence:', forecast.methodology.confidence);

// Usage - Linear Regression
const trendForecast = await getRewardsForecast(
  'he1iusunGwqrNtafDtLdhsUQDFvo13z9sUa36PauBtk',
  {
    forecastPeriod: '90d',
    lookbackDays: 60,
    model: 'linear_regression'
  }
);

console.log('Optimistic APY:', trendForecast.projections.optimistic.apy_pct, '%');
console.log('R-squared:', trendForecast.methodology.r_squared);
```

```python Python
import requests

def get_rewards_forecast(
    vote_account: str,
    forecast_period: str = "30d",
    lookback_days: int = 30,
    model: str = "rolling_average",
    include_breakdown: bool = False
):
    base_url = "http://validator-api.helius-internal.com"
    

    params = {
        "forecast_period": forecast_period,
        "lookback_days": lookback_days,
        "model": model,
        "include_breakdown": str(include_breakdown).lower(),
    }

    url = f"{base_url}/validators/{vote_account}/rewards/forecast"
    response = requests.get(url, params=params)
    return response.json()

# Usage - Rolling Average
forecast = get_rewards_forecast(
    "he1iusunGwqrNtafDtLdhsUQDFvo13z9sUa36PauBtk",
    forecast_period="30d",
    lookback_days=30,
    model="rolling_average",
    include_breakdown=True
)

print(f"Baseline APY: {forecast['projections']['baseline']['apy_pct']}%")
print(f"Baseline SOL: {forecast['projections']['baseline']['rewards']['total_sol']}")
print(f"Confidence: {forecast['methodology']['confidence']}")

# Usage - Linear Regression
trend_forecast = get_rewards_forecast(
    "he1iusunGwqrNtafDtLdhsUQDFvo13z9sUa36PauBtk",
    forecast_period="90d",
    lookback_days=60,
    model="linear_regression"
)

print(f"Optimistic APY: {trend_forecast['projections']['optimistic']['apy_pct']}%")
print(f"R-squared: {trend_forecast['methodology']['r_squared']}")
```

## Response Schema

<ResponseField name="vote_account" type="string">
  The validator's vote account address
</ResponseField>

<ResponseField name="forecast_period_days" type="integer">
  Number of days forecasted into the future
</ResponseField>

<ResponseField name="projections" type="object">
  Three scenario projections with varying levels of optimism

  <Expandable title="projections properties">
    <ResponseField name="conservative" type="object">
      Conservative scenario (mean - 1 std dev or lower confidence bound)

      <Expandable title="conservative properties">
        <ResponseField name="apy_pct" type="number">
          Gross APY (all rewards / stake, before commission deductions)
        </ResponseField>

        <ResponseField name="staker_apy_pct" type="number">
          Staker APY (what delegators receive after commissions)
        </ResponseField>

        <ResponseField name="rewards" type="object">
          Projected rewards breakdown

          <Expandable title="rewards properties">
            <ResponseField name="inflation_rewards_lamports" type="integer">
              Projected inflation rewards in lamports
            </ResponseField>

            <ResponseField name="inflation_rewards_sol" type="number">
              Projected inflation rewards in SOL
            </ResponseField>

            <ResponseField name="mev_rewards_lamports" type="integer">
              Projected MEV rewards in lamports
            </ResponseField>

            <ResponseField name="mev_rewards_sol" type="number">
              Projected MEV rewards in SOL
            </ResponseField>

            <ResponseField name="block_rewards_lamports" type="integer">
              Projected block rewards in lamports
            </ResponseField>

            <ResponseField name="block_rewards_sol" type="number">
              Projected block rewards in SOL
            </ResponseField>

            <ResponseField name="total_lamports" type="integer">
              Total projected rewards in lamports
            </ResponseField>

            <ResponseField name="total_sol" type="number">
              Total projected rewards in SOL
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="baseline" type="object">
      Baseline scenario (mean or median projection)

      Same structure as `conservative`
    </ResponseField>

    <ResponseField name="optimistic" type="object">
      Optimistic scenario (mean + 1 std dev or upper confidence bound)

      Same structure as `conservative`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="methodology" type="object">
  Detailed information about the forecasting methodology

  <Expandable title="methodology properties">
    <ResponseField name="lookback_days" type="integer">
      Number of historical days analyzed
    </ResponseField>

    <ResponseField name="epochs_analyzed" type="integer">
      Number of epochs included in analysis
    </ResponseField>

    <ResponseField name="confidence" type="string">
      Confidence level of the forecast

      Values: `low`, `medium`, `high`
    </ResponseField>

    <ResponseField name="confidence_interval_pct" type="number">
      Confidence interval percentage (typically 95%)
    </ResponseField>

    <ResponseField name="method" type="string">
      The forecasting method used

      Values: `rolling_average`, `linear_regression`
    </ResponseField>

    <ResponseField name="apy_std_dev_pct" type="number" nullable>
      Standard deviation of APY (rolling_average only)
    </ResponseField>

    <ResponseField name="slope_pct_per_epoch" type="number" nullable>
      Trend slope per epoch (linear_regression only)
    </ResponseField>

    <ResponseField name="intercept_pct" type="number" nullable>
      Y-intercept of regression line (linear_regression only)
    </ResponseField>

    <ResponseField name="r_squared" type="number" nullable>
      R² goodness of fit (linear_regression only)

      Range: 0-1, where 1 is perfect fit
    </ResponseField>

    <ResponseField name="standard_error_pct" type="number" nullable>
      Standard error of the estimate (linear_regression only)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="query_timestamp" type="integer">
  Unix timestamp when the forecast was generated
</ResponseField>

## Understanding Confidence Levels

| Confidence | Criteria | Interpretation |
|------------|----------|----------------|
| **High** | Low variance, sufficient data, strong fit | Forecast is reliable for planning |
| **Medium** | Moderate variance or limited data | Use for general guidance |
| **Low** | High variance, insufficient data, or poor fit | Forecast has significant uncertainty |

## Note

<Warning>
**Important Limitations:**
- Forecasts assume historical patterns continue
- Cannot predict sudden network changes (inflation adjustments, MEV shocks)
- More accurate for shorter forecast periods
- Always consider external factors (network upgrades, market conditions)
</Warning>
