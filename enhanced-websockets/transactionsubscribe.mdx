---
title: "transactionSubscribe"
description: "Subscribe to receive real-time transaction events based on specified filters."
---

## Endpoints

Enhanced websockets are currently available on mainnet and devnet with the following URLs:

- **Mainnet** `wss://atlas-mainnet.helius-rpc.com/?api-key=<API_KEY>`
- **Devnet** `wss://atlas-devnet.helius-rpc.com/?api-key=<API_KEY>`

<Note>Websockets have a 10-minute inactivity timer; implementing health checks and sending pings every minute is heavily recommended to keep the websocket connection alive.</Note>

## Authorizations

<ParamField query="api-key" type="string" required>
  Your Helius API key. You can get one for free in the [dashboard](https://dashboard.helius.dev/api-keys).
</ParamField>

## Body

<ParamField body="params" type="array" required>
    <Expandable title="properties" defaultOpen>
        <ParamField body="TransactionSubscribeFilter" type="object" required>
            <Expandable title="filter properties">
                <ParamField body="vote" type="boolean">
                    A boolean flag to include/exclude vote-related transactions.
                </ParamField>
                
                <ParamField body="failed" type="boolean">
                    A boolean flag to include/exclude transactions that failed.
                </ParamField>
                
                <ParamField body="signature" type="string">
                    Filters updates to a specific transaction based on its signature.
                </ParamField>
                
                <ParamField body="accountInclude" type="array">
                    A list of accounts for which you want to receive transaction updates. This means that only one of the accounts must be included in the transaction updates (e.g., Account 1 OR Account 2).
                </ParamField>
                
                <ParamField body="accountExclude" type="array">
                    A list of accounts you want to exclude from transaction updates.
                </ParamField>
                
                <ParamField body="accountRequired" type="array">
                    Transactions must involve these specified accounts to be included in updates. This means that all of the accounts must be included in the transaction updates (e.g., Account 1 AND Account 2).
                </ParamField>
            </Expandable>
        </ParamField>

        <ParamField body="TransactionSubscribeOptions" type="object">
            <Expandable title="options properties">
                <ParamField body="commitment" type="string">
                    Defines the commitment level for the transaction. Supported values: **finalized**, **confirmed**, **processed** (default is **finalized**).
                </ParamField>
                
                <ParamField body="encoding" type="string">
                    Specifies the format for transaction data. Supported values: **base58**, **base64**, **base64+zstd**, **jsonParsed** (default is **jsonParsed**).
                </ParamField>
                
                <ParamField body="transactionDetails" type="string">
                    Level of detail to include in transaction data. Options: **full**, **accounts**, **signatures**, **none** (default is **full**).
                </ParamField>
                
                <ParamField body="showRewards" type="boolean">
                    Whether to include rewards in transaction data. Default is **true**.
                </ParamField>
                
                <ParamField body="maxSupportedTransactionVersion" type="integer">
                    Maximum transaction version to support. Default is **0**.
                </ParamField>
            </Expandable>
        </ParamField>
    </Expandable>
</ParamField>

## Response

<ResponseField name="result" type="integer">
  Subscription id (needed to unsubscribe)
</ResponseField>
