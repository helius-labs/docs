<Accordion title="为什么我会收到 401 错误？">
  
  ## 这意味着什么
  <Warning>**401 未授权** - 您的请求缺少有效的身份验证凭据。</Warning>
  
  ## 常见原因
  - API 密钥无效或缺失
  - API 密钥放置在错误的位置
  - 访问控制规则阻止了您的请求
  - API 密钥已过期或被撤销
  
  ## 解决方案
  
  1. **验证您的 API 密钥格式**

     ```
     https://mainnet.helius-rpc.com/?api-key=YOUR_API_KEY
     ```

  
  2. **检查 API 密钥位置**
     - 确保 API 密钥在查询参数中
     - 确认没有多余的空格或字符
  
  3. **查看访问控制规则**
     - 检查您的 [仪表板设置](https://dashboard.helius.dev/) 中的 IP 限制
     - 如果使用浏览器请求，请确保您的域名在允许列表中
  
  <Info>有关详细的身份验证设置，请参阅我们的 [身份验证](/zh/api-reference/authentication) 指南。</Info>
</Accordion>

<Accordion title="为什么我会收到429错误？">
  
  ## 这意味着什么
  <Warning>**429 请求过多** - 您已超出计划的速率限制。</Warning>
  
  ## 常见原因
  - 发出请求的速度超过了您的计划允许的速度
  - 突发流量超过瞬时限制
  - 多个应用程序共享同一个API密钥
  - 低效的代码导致冗余请求
  
  ## 解决方案
  
  1. **监控您的使用情况**
     - 查看您的[仪表板](https://dashboard.helius.dev/usage)中的`Rate Limited Requests`图表
     - 检查哪些端点达到了限制
  
  2. **优化您的请求**
     - 尽可能缓存响应
     - 将多个操作合并为单个调用
     - 删除不必要的轮询或重复请求
  
  3. **实施速率限制**
     - 在您的应用程序中添加请求之间的延迟
     - 对重试使用指数退避
  
  4. **考虑升级**
     - 查看更高等级的[计划和速率限制](/zh/billing/plans)
  
  <Tip>速率限制每分钟重置一次，因此临时限制通常会很快解决。</Tip>
</Accordion>

<Accordion title="为什么我会收到500错误？">
  
  ## 这意味着什么
  <Warning>**500 内部服务器错误** - 在处理您的请求时发生了服务器端错误。</Warning>
  
  ## 常见原因
  - 请求负载格式错误
  - 服务器遇到临时问题
  - 无效参数导致服务器错误
  - 网络连接问题
  
  ## 解决方案
  
  1. **验证您的请求**
     - 确保JSON负载格式正确
     - 验证是否包含所有必需参数
     - 检查参数类型是否与API规范匹配
  
  2. **检查服务状态**
     - 访问 [Helius 状态页面](https://helius.statuspage.io/) 查看正在进行的问题
     - 查找任何报告的中断或性能下降
  
  3. **实施重试逻辑**
     - 等待几秒钟后再重试
     - 对多次尝试使用指数退避
  
  4. **获取支持**
     - 如果错误持续，请联系支持并提供您的请求详细信息
     - 包括确切的请求负载和时间戳
  
  <Note>服务器错误通常是暂时的，通常会自动解决。</Note>
</Accordion>

<Accordion title="为什么我会收到503错误？">
  
  ## 这意味着什么
  <Warning>**503 服务不可用** - 服务器暂时过载或正在维护。</Warning>
  
  ## 常见原因
  - 高流量导致临时过载
  - 计划的维护窗口
  - 达到服务器容量限制
  - 网络基础设施问题
  
  ## 解决方案
  
  1. **等待并重试**
     - 等待30-60秒后再试
     - 此错误通常会随着负载平衡而解决
  
  2. **实施智能重试**
     - 使用指数退避（从1秒开始，然后是2秒，4秒等）
     - 设置最大重试限制（3-5次尝试）
     - 添加抖动以避免雷鸣般的羊群效应
  
  3. **检查维护情况**
     - 查看 [Helius 状态页面](https://helius.statuspage.io/) 了解计划的维护
     - 根据宣布的维护窗口进行计划
  
  4. **分配负载**
     - 如果可能，分散请求时间
     - 避免可能触发过载保护的突发模式
  
  <Tip>503错误旨在是暂时的 - 随着服务器负载的减少，服务会自动恢复。</Tip>
</Accordion>

<Accordion title="为什么我会收到 504 错误？">
  
  ## 这意味着什么
  <Warning>**504 Gateway Timeout** - 服务器在超时时间内没有收到上游服务的响应。</Warning>
  
  ## 常见原因
  - 网络连接问题
  - 复杂操作超过超时限制
  - 网络拥堵严重时区块链响应缓慢
  - 大量数据请求处理时间过长
  
  ## 解决方案
  
  1. **检查您的连接**
     - 确保您的互联网连接稳定
     - 通过简单请求测试以排除本地问题
  
  2. **优化大请求**
     - 将大型批量请求分成较小的部分
     - 对数据量大的查询使用分页
     - 考虑使用 WebSocket 连接以获取实时数据
  
  3. **实施超时**
     - 在客户端代码中设置适当的超时值（30-60 秒）
     - 优雅地处理超时错误并重试
  
  4. **监控服务状态**
     - 查看 [Helius 状态页面](https://helius.statuspage.io/) 以了解网络问题
     - 查找高区块链拥堵的报告
  
  <Note>网关超时通常表示网络拥堵或复杂操作。考虑将大请求分成较小的部分。</Note>
</Accordion>
