## 将交易类型映射到来源
每种交易类型由特定的程序来源生成。理解这些关系有助于识别哪些程序正在触发特定的交易类型。

下面的映射展示了来源（程序）与特定交易类型之间的关系：

- **类型到来源**：显示哪些程序生成特定的交易类型
- **来源到类型**：显示特定程序可以生成的所有交易类型

例如，**NFT_GLOBAL_BID** 交易由 **MAGIC_EDEN** 或 **SOLANART** 生成，而 **CANDY_MACHINE_V2** 专门生成 **NFT_MINT** 事件。

```
## Using Transaction Types in Your Code
When implementing webhook listeners, you can filter for specific transaction types of interest.

Here's a simple example of how to listen for NFT sale events:

```

```

javascript
// 用于 NFT 销售的示例 webhook 处理程序
app.post('/webhook', (req, res) => {
  const event = req.body;
  
  // 过滤 NFT 销售交易
  if (event.type === 'NFT_SALE') {
    console.log('检测到 NFT 销售！');
    console.log('卖家:', event.seller);
    console.log('买家:', event.buyer);
    console.log('价格:', event.amount);
    console.log('NFT:', event.nft?.mint);
    
    // 处理销售事件
    // ...
  }
  
  res.status(200).send('OK');
});
```

您可以使用 Type-to-Source 和 Source-to-Type 映射来预测哪些程序将生成特定的交易类型。
