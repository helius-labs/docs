## 为什么使用事件监听？

- **实时更新**：在区块链事件发生时即时接收通知
- **降低基础设施成本**：无需持续轮询
- **可扩展架构**：高效处理大量事件
- **灵活的事件过滤**：精确配置您想要监控的事件

## 可用的事件监听选项

### Webhooks

[Webhooks](/zh/webhooks) 是 HTTP 回调，当 Solana 区块链上发生事件时将数据传递到您的应用程序。它们非常适合：

- 跟踪代币转移
- 监控 NFT 活动
- 检测程序交互
- 接收账户变更通知

Helius webhooks 提供全面的事件数据和详细的交易信息，使您能够轻松构建对链上活动作出响应的应用程序。

### Websockets

[Websockets](/zh/rpc/websocket) 提供您的应用程序与 Solana 区块链之间的持久连接，实现实时的双向通信。它们特别适用于：

- 需要即时更新的应用程序
- 需要实时反映区块链状态的用户界面
- 对低延迟要求较高的场景
- 需要持续数据流的情况

与 webhooks 不同，websockets 保持开放连接，消除了每次事件传递的 HTTP 开销，提供更快的通知。

## 在 Webhooks 和 Websockets 之间进行选择

- **使用 Webhooks 的场景**：需要可靠的交付和重试机制，偏好无状态方法，或希望异步处理事件。

- **使用 Websockets 的场景**：需要最低的延迟，要求双向通信，或正在构建高度交互的应用程序。

## 使用场景

- **NFT 市场**：实时跟踪上架、销售和转移
- **钱包**：通知用户收到的交易
- **DeFi 应用**：监控交易、流动性变化和协议事件
- **游戏**：捕获存储在链上的游戏内操作
- **分析平台**：实时聚合交易数据
