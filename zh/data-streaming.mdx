## 什么是 Solana 数据流？

数据流允许应用程序在 Solana 区块链上发生事件时接收实时更新。与其反复轮询更新，不如建立持久连接，当交易被处理、账户发生变化或区块生成时，立即将数据推送到您的应用程序。

这对于需要秒级数据的应用程序至关重要，例如：

- **交易应用程序** 监控价格变化和清算
- **DeFi 协议** 跟踪用户交互和状态变化
- **NFT 市场** 检测销售、上市和转移
- **分析平台** 收集全面的区块链指标
- **钱包** 显示实时余额和交易更新

## Helius 流解决方案

我们提供多种流选项，以满足您应用程序的特定需求，从简单的 WebSocket 连接到用于算法交易操作的超低延迟原始数据流。

### Shred Delivery (Beta)

**超低延迟原始数据流**

<Card title="Shred Delivery" icon="bolt" href="/zh/shred-delivery">
  提供原始 Solana shreds 的顶级低延迟服务
</Card>

**关键特性：**
- **最早访问**：原始 shreds 在网络生成时即被传送
- **未处理数据**：原始格式允许根据您的交易策略进行定制处理
- **验证者优势**：Helius 运营 Solana 上最高质押的验证者，以最小延迟接收 shreds
- **白手套供应**：精心管理的访问，提供专门的工程支持
- **仅限 Beta 访问**：目前仅限于合格的交易团队和算法交易操作

**最佳适用对象：** 高频交易台、套利交易者、清算系统以及其他对延迟敏感的策略，其中毫秒决定盈利能力

### LaserStream（开发者+ Devnet，专业+ Mainnet）

**具有高级功能的下一代流媒体**

<Card title="LaserStream" icon="rocket" href="/zh/laserstream">
  使用 gRPC 协议的高性能流媒体，历史重播和多节点可靠性
</Card>

**关键特性：**
- **历史重播**：自动回填多达 3000 个插槽（约 20 分钟）的丢失数据
- **多协议支持**：gRPC 现已可用，WebSockets 即将推出
- **自动重连**：内置连接管理，具有智能重试逻辑
- **全球端点**：在全球 7 个以上地区可用，以实现最佳延迟
- **即插即用替代**：兼容现有的 Yellowstone gRPC 实现

**最佳适用对象：** 后端服务、高吞吐量应用程序、需要保证数据传输的关键任务系统

### 增强型 WebSockets（商业+）

**具有高级过滤功能的高性能 WebSockets**

<Card title="Enhanced WebSockets" icon="signal-stream" href="/zh/enhanced-websockets">
  更快的 WebSocket 连接，具有增强的账户和交易订阅
</Card>

**关键特性：**
- **更快的响应时间**：优化以实现比标准 WebSockets 更低的延迟
- **增强过滤**：高级订阅选项以精确定位数据
- **账户和交易订阅**：监控特定账户或交易模式
- **Atlas 基础设施**：由我们的高性能流媒体后端提供支持

**最佳适用对象：** 实时前端应用程序、中等量后端、自定义过滤需求

### 标准 WebSockets

**与 Solana 兼容的 WebSocket API，具有 Helius 的可靠性**

<Card title="Standard WebSockets" icon="wifi" href="/zh/rpc/websocket">
  标准 Solana WebSocket 方法，增强了 Helius 基础设施的可靠性
</Card>

**关键特性：**
- **完全兼容性**：适用于任何 Solana WebSocket 客户端库
- **标准方法**：支持所有 Solana 订阅类型（账户、程序、日志、签名、插槽）
- **多网络**：可用于主网和开发网
- **Helius 可靠性**：在标准 RPC 节点上增强的正常运行时间和性能

**最佳适用场景：** 现有应用程序、标准用例、广泛的兼容性要求

### Webhooks

**事件驱动的通知传递到您的端点**

<Card title="Webhooks" icon="webhook" href="/zh/webhooks">
  针对特定链上活动的服务器到服务器事件通知
</Card>

**关键特性：**
- **解析的事件数据**：可读性强的交易数据用于销售、交换等
- **多种类型**：增强型、原始型和 Discord webhook 选项
- **交易过滤**：订阅特定事件类型和地址
- **可靠的交付**：自动重试和交付确认

**最佳适用场景：** 事件驱动的架构、通知系统、与外部服务的集成
